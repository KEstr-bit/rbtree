# Красно-чёрное дерево и Map на C

Проект содержит реализацию обобщённого красно-чёрного дерева и ассоциативного массива поверх него. Данные хранятся как void *, поэтому дерево может работать с разными типами ключей и значений при передаче пользовательских функций сравнения и освобождения памяти.

## Состав проекта

| Файл | Назначение |
| --- | --- |
| `rbtree.h` | Публичный интерфейс красно-чёрного дерева |
| `rbtree.c` | Реализация дерева: вставка, удаление, поиск, обход, `join`, `split` |
| `rbmap.h` | Публичный интерфейс ассоциативного массива |
| `rbmap.c` | Обёртка `Map` над `RBTree` |
| `main.c` | Демонстрационные тесты и примеры использования |

## Краткое описание решения

### RBTree

`RBTree` — самобалансирующееся бинарное дерево поиска. Для сохранения баланса после операций используются стандартные свойства красно-чёрного дерева:

- каждый узел имеет цвет: красный или чёрный;
- корень всегда чёрный;
- Общий-узел `nil` используется вместо `NULL`-листьев;
- после вставки и удаления выполняется восстановление свойств дерева через повороты и перекраску.

Поддерживаемые операции:

- создание и уничтожение дерева;
- вставка пары `key/value`;
- обновление значения при повторной вставке существующего ключа;
- удаление по ключу;
- поиск значения по ключу;
- проверка наличия ключа;
- получение минимального и максимального узла;
- очистка дерева;
- получение размера;
- inorder-обход через итератор;
- объединение деревьев через `rbtree_join()`;
- разделение дерева через `rbtree_split()`.

> В этой реализации `rbtree_split(tree, key, &left, &right)` оставляет ключи `<= key` в левом дереве, а ключи `> key` — в правом.

### Map

`Map` — простой ассоциативный массив, использующий `RBTree` внутри. Он предоставляет более удобные функции:

- `map_put()` — добавить или обновить значение;
- `map_get()` — получить значение по ключу;
- `map_remove()` — удалить ключ;
- `does_map_contain()` — проверить наличие ключа;
- `get_map_size()` — получить количество элементов;
- `clear_map()` — очистить контейнер.

Дерево и map владеют переданными ключами и значениями. Для корректного освобождения памяти при создании нужно передать функции `free_key` и `free_value`.

## Требования

- компилятор C: `gcc` или `clang`;
- стандарт C11 или совместимый.

Дополнительные библиотеки не требуются.

## Сборка и запуск тестов

В проекте нет отдельного unit-test-фреймворка: проверки находятся в `main.c` и запускаются как обычная программа.

Перейдите в каталог с исходными файлами и выполните:

```bash
gcc -std=c11 -Wall -Wextra -pedantic -g main.c rbtree.c rbmap.c -o rb_test
./rb_test
```

Для `clang` команда аналогична:

```bash
clang -std=c11 -Wall -Wextra -pedantic -g main.c rbtree.c rbmap.c -o rb_test
./rb_test
```

## Что проверяет `main.c`

Тестовая программа демонстрирует и проверяет:

1. вставку строковых ключей в разном порядке;
2. корректность балансировки дерева после вставок;
3. поиск существующих и отсутствующих ключей;
4. получение минимального и максимального ключа;
5. обновление значения при повторной вставке ключа;
6. удаление узлов;
7. работу с пустым деревом и деревом из одного узла;
8. массовую вставку целых ключей `0..19` и удаление чётных;
9. inorder-обход, который должен выводить ключи по возрастанию;
10. операции `rbtree_join()` и `rbtree_split()`;
11. работу `Map`: `put`, `get`, `remove`, `contains`, `min`, `max`;
12. массовую вставку 100 ключей в `Map` и удаление половины.

Успешный запуск завершается кодом `0` и выводит демонстрационный лог, начинающийся с раздела `ДЕРЕВО`, а затем раздел `MAP`.
